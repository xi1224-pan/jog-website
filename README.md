# 跑步數位看板 - 全棧雲端化路跑科學分析系統

本專案是一個**產品級的全棧（Full-Stack）運動科學大數據看板**。打破傳統單機網頁（`localhost`）與單機資料庫（SQLite/MySQL Workbench）的裝置限制，將後端微服務與分散式資料庫完全部署於雲端環境，實現 PC 紀錄、筆電報告、手機隨時查看的「多端實時同步」與「零信任（Zero Trust）資安防護」。

---

## 🛠️ 技術棧與三層式雲端架構 (System Architecture)

本系統採用現代軟體工程主流的解耦架構，將數據採集、業務邏輯與持久化儲存完全分離：

1. **前端展示層 (Frontend Brain)**
   * **技術：** HTML5 / CSS3 (響應式 Apple 風格設計) / 原生 JavaScript (ES6+)。
   * **功能：** 透過非同步 **Fetch API** 與後端網關通訊，利用 **Chart.js** 進行多維度計量學運算（如 Pearson 相關係數）與實時動態圖表渲染。
2. **後端微服務網關 (Backend API Gateway)**
   * **技術：** Python / **FastAPI** 框架。
   * **部署平台：** **Render** 雲端伺服器 (24/7 在線 RESTful API)。
   * **特性：** 利用 FastAPI 的高併發（High Concurrency）非同步處理能力，內建中間件處理跨來源資源共享（CORS）安全限制。
3. **雲端數據持久化層 (Cloud Database)**
   * **技術：** 分散式雲端資料庫 **Neon PostgreSQL**。
   * **特性：** 取代本地端單機 SQLite，透過 SSL 加密通道與雲端連線池管理，保障路跑核心資產的永久安全儲存。

## ✨ 核心數據科學功能 (Key Features)

* **📅 智慧型動態日曆控制台：** 自動非同步異步檢索雲端數據，比對有紀錄之訓練日並自動渲染「橘色數據小圓點」；具備狀態機邏輯，智慧切換「唯讀瀏覽」與「編輯儲存」模式，防止資料覆蓋失真。
* **📊 Pearson 相關係數 ($r$ 值) 實時監測：** 內建統計學數學模型，動態計算「溫濕度體感指數 vs 想吐程度（乳酸閾值不適感）」、「氣溫 vs 平均速度」等五大維度散佈圖與線性迴歸趨勢線，利用數學實時量化環境特徵對身體的實質權重影響。
* **⏱️ 速耐力衰退率分析引擎：** 實作分段配速運算邏輯，針對非整數公里進行「尾數標準化還原」；自動計算各公里間的掉速百分比，當後半段總衰退率超過 15% 臨界值時，前端自動觸發警告標籤，精準揪出體能瓶頸。
* **📈 組別雙模序列折線圖：** 提供選單動態切換「週三訓練組（週間團隊拉扯）」與「週日自主組（週末長距離基礎耐力）」，X 軸嚴謹遵循時間序列推進，Y 軸解碼算式自動將浮點數分鐘還原為標準路跑耗時戳（如 `11:42`）。
* **🗂️ 雙重數據極端樞紐表：** 自動清洗歷史數據分流，即時運算兩大組別的累積里程、均速與累積熱量；並智慧排序出 Top 2 速度極值表與最不適（想吐 Top 3）環境氣象對照表。

## 🔒 環境變數與資安最佳實務 (Security Best Practices)

本專案落實**原始碼與敏感憑證完全分離**的企業級資安標準，源碼公開於 GitHub 亦具備零信任架構安全性：

### 1. 雲端環境變數隱藏 (Environment Variables)
後端 `jog.py` 拒絕硬編碼（Hardcoding）任何敏感金鑰，核心憑證完全鎖在 Render 雲端控制台（Environment）中，透過作業系統環境變數動態讀取：
* `DATABASE_URL`：Neon PostgreSQL 雲端資料庫密鑰連線字串。
* `ADMIN_PASSWORD`：自訂的管理員身份驗證密碼。

### 2. 運行時動態身份校驗 (Runtime Authentication)
前端網頁（`運動數據.html`）在執行敏感之數據變更（POST 儲存/修改、DELETE 刪除）時，不將密碼寫死在網頁中，而是採用 `prompt()` 於**運行時動態彈出提示框手動輸入**。密碼將透過 `X-Password` 自訂標頭（Header）傳送至 FastAPI，與雲端環境變數進行比對校驗，無權限之惡意請求將直接被網關攔截，拒絕連線修改。

## 🚀 部署與本地運行指南 (Deployment & Usage)

### 後端雲端配置
1. 將本儲存庫之 `jog.py` 與 `requirements.txt` 部署至 Render Web Service。
2. 於 Render 後台 **Advanced** ➔ **Environment Variables** 填入上述之 `DATABASE_URL` 與 `ADMIN_PASSWORD`。

### 前端本地操作
1. 本地免安裝任何後端環境與資料庫軟體，直接雙擊打開 **`運動數據.html`**。
2. 網頁會自動跨海連線至雲端 API 並渲染全局歷史大數據。
3. 當點擊有圓點之歷史日期時，可查閱同天歷史脈絡對比；若欲點選空日期新增當日數據，在點選確認儲存時，請於彈出視窗輸入您在 Render 後台設定的 `ADMIN_PASSWORD` 即可安全通車！
